一个很小，但能把 agent 味道讲清楚的项目

它会先把目标拆开，再去扫工作区找线索，然后生成一份能直接贴出去的 Markdown 报告，还会顺手做一次自检。

看项目说明 看提交模板

它到底做什么

  • 先把一句话目标拆成 4 到 5 个步骤
  • 再去扫目标目录，找最像“证据”的文件
  • 然后生成一份可直接展示的 Markdown 报告
  • 最后再检查一遍结果有没有漏项

你可以怎么讲

这不是那种“只会写一句我用了 AI”的项目，而是有点像真的 agent 工作流： 会规划，会看上下文，会产出结果，也会检查自己。

提交时可以直接写的口语版内容

上传材料怎么配

  • 终端运行截图，能看到命令和输出
  • `output/report.md`，证明它真的产出了结果
  • `output/trace.json`，证明它有结构化输出
  • GitHub 仓库链接，证明项目是完整的
